# Research Session: When Code Could Clone Itself (Chapter 2)

## Context
This is research for Chapter 2 of "The Freedom Paradox." This chapter uses John O'Nolan's February 2026 newsletter as a jumping-off point: AI can now clone entire open-source codebases in days, potentially making software licenses meaningless.

## What I Need

### The Cloudflare Incident
- Cloudflare cloning a competitor's (Vercel's) open-source Next.js codebase using AI in one week
- Steve Faulkner's blog post: "How we rebuilt Next.js with AI in one week"
- The product "vinext" — what it does, performance claims (4x faster builds, 57% smaller bundles)
- Vercel/Next.js community response
- Legal analysis: does AI-assisted rewriting from scratch violate the original license?

### Simon Willison's Precedent
- December 2025 experiment porting an HTML parsing library from Python to JavaScript using AI
- The legal/ethical questions he raised
- Community discussion that followed

### The "Studio Ghibli Moment"
- O'Nolan's analogy to the AI art controversy
- The parallel: when anyone can replicate creative/technical work with AI, what happens to the original creators?

### Reverse Engineering Closed Software
- Geoffrey Huntley's "z80 technique" revealing Atlassian's Rovo AI assistant source code
- LLMs' capability for reverse-engineering proprietary software
- O'Nolan's prediction: what takes a week with open source will soon take a week with closed source too

### The License Question
- If AI can rewrite code from scratch based on an open-source codebase, does the original license apply?
- Legal precedent (if any) for AI-generated derivatives
- Ghost's own experience with Substack copying code
- The broader question: do software licenses mean anything in an age of AI?

### O'Nolan's Paradox
- "AI might end up fulfilling [Stallman's] vision more completely than open source ever did"
- The irony: AI makes ALL software effectively "open" (can be studied, replicated, modified) regardless of license
- What this means for Ghost's own open-source model
